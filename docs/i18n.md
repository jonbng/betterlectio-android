# Internationalization (i18n)

BetterLectio supports **Danish** (default) and **English**.

## Resource layout

| Path | Role |
|------|------|
| `app/src/main/res/values/strings.xml` | Danish default (fallback) |
| `app/src/main/res/values-en/strings.xml` | English |
| `app/src/main/res/xml/locales_config.xml` | System “App language” (API 33+) |

**Rule:** every `name=` in the default file must exist in `values-en/` and vice versa.  
Enforced by `I18nStringParityTest`.

## Language preference

- Settings → **Sprog / Language**: System | Dansk | English
- Stored in `SettingsStore` as `AppLanguage`
- Applied via `AppLocale.apply()` → `AppCompatDelegate.setApplicationLocales`
- Applied at process start from `MainActivity`

## How to add user-facing text

| Layer | Pattern |
|-------|---------|
| Compose UI | `stringResource(R.string.foo)` |
| ViewModels | `UiText.Res(R.string.foo)` → resolve with `asString()` / `resolve(context)` |
| Workers / notifications / widgets | `context.getString(R.string.foo, …)` |
| Lectio HTML parsers | **Do not** use `strings.xml`. Keep Danish protocol tokens (e.g. `"Hele dagen"`). |
| Timber / logs | English OK; not user-facing |

Helper: `dk.betterlectio.android.core.i18n.UiText`

```kotlin
// ViewModel
message = UiText.Res(R.string.private_event_created)

// Compose
Text(state.message?.asString().orEmpty())
```

## Display dates

Use `Locale.getDefault()` for user-facing formatters (after locales are applied).  
Lectio **parsing** stays on a fixed Danish/ROOT locale (`LectioDateUtils` parse paths).

## Follow-up agent packages

1. Compose UI leftovers (hardcoded labels, contentDescriptions)
2. Remaining ViewModel / snackbar strings via `UiText`
3. Notifications + live lesson
4. Glance widgets
5. Domain → UI labels (`ScheduleEvent.timeLabel` / `statusLabel`)
6. Update probe / misc feature messages
7. Plurals + accessibility audit

Each package: add DA key → EN key → wire call sites → run parity test.
